# Dynamic tension-reframe sweep — which tensions reframe when the system is made dynamic

## The one structural lever this whole sweep turns on

The two GU councils (`council-inherited-tension-resolution`, `council-trit-resolution`)
did not merely grade the ~20 inherited tensions COMPLETE-INHERIT. They proved *why*, and the
"why" is a **type theorem**, stated in the trit-council's own words:

> "a discrete label carries no scale (⇒ no magnitude), no measure/factorization (⇒ no
> dynamical basis), and no dynamics (⇒ no non-perturbative content) ... `sigma` : sign ::
> `tau` : count — two discrete labels, two discrete kinds, and **nothing** whose content is a
> **magnitude**, a **basis**, or a **non-perturbative dynamics**."

That is a wall for **static discrete data**. It is *not* a wall for physics. Read it as a
**type signature of what is missing**, and it hands DU its target list precisely:

| The static data provably CANNOT reach | because it lacks | which is exactly the type of a … |
|---|---|---|
| a **MAGNITUDE** (CC 10⁻¹²⁰, hierarchy, mass ratios, entropy scale) | a **scale** | **flow / process** (a roll that runs sets a scale: `Λ~1/√N`) |
| a **DYNAMICAL BASIS** (measurement, Born, preferred-basis, split) | a **measure / factorization / selection** | **measure + closure/fixed-point** (μ prices histories; finality selects) |
| a **NON-PERTURBATIVE dynamics** (confinement, singularities) | a **dynamics** | **algebra of records with growth** (emergent continuum) |

The trit-council itself names the escape hatch — "Closing them requires the continuous,
measure-theoretic, dynamical object every leg waits on: the **B5 source action**." DU's thesis
is that the dynamic layer (records / finality / issuance / the roll / the record-algebra) *is*
a continuous, measure-theoretic, dynamical object — a **different type** from a Z/2 or a Z/3.
So the honest question of this sweep is **not** "can dynamics help?" (it can, generically —
that is the hand-wave the charter forbids). It is: **for each tension, is there a CONCRETE DU
dynamical object of the RIGHT TYPE with a named mechanism, or only a type-match with no
construction?** Right-type-with-mechanism ⇒ **REFRAMES-DYNAMICALLY**. Right-type,
mechanism-class named but no DU construction ⇒ **PARTIAL**. No DU object reaches the sector, or
the sector is already dynamical and still unsolved ⇒ **STILL-INHERITED**.

The known template: the transducer reframes the DE magnitude — a `RESOLVED_NO_GO` for the
static data (σ fixes only the DE *sign*) — into `Λ~1/√N`, the readout of *how far the roll has
run*. The magnitude stops being a posited constant and becomes a **dynamical variable's
value**. That is the type-shift. This sweep asks the same of every row.
